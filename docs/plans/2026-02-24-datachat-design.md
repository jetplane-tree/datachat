# DataChat - AI 数据对话分析平台

## 项目定位

一个自然语言驱动的数据分析平台。用户选择内置数据集或上传自己的数据，用中文提问，AI 自动生成 SQL 查询、可视化图表和分析洞察。

**目标用户**：业务方 / 数据分析师 / 面试官体验
**一句话介绍**：用自然语言和数据对话，让数据分析像聊天一样简单。

## 核心用户流程

1. 进入首页 → 选择内置数据集 或 上传 CSV/Excel
2. 数据预览 → 自动展示数据概况（行列数、字段类型、示例数据）
3. 对话分析 → 用自然语言提问（如"各月销售额趋势是什么？"）
4. AI 响应 → 生成 SQL + 可视化图表 + 文字洞察，以卡片形式展示
5. 继续追问 → 支持多轮对话上下文
6. 导出 → 分析历史可查看

## 技术架构

```
┌── 前端（浏览器）──────────────────────┐
│  React + Vite + Tailwind + shadcn/ui  │
│  ECharts 渲染图表                      │
│  DuckDB WASM 执行 SQL 查询            │
└──────────┬────────────────────────────┘
           │ 用户提问 + 数据 schema
           ▼
┌── Vercel API Routes（后端）───────────┐
│  接收用户问题 + 表结构信息              │
│  调用 LLM API（API Key 安全存储）      │
│  返回：SQL + 图表配置 + 洞察文字        │
└───────────────────────────────────────┘
```

### 技术栈

| 层 | 技术 | 说明 |
|---|------|------|
| 前端框架 | Next.js (App Router) | 全栈框架，前后端一体 |
| UI | Tailwind CSS + shadcn/ui | 现代、可定制、去模板感 |
| 图表 | ECharts | 中文友好，图表类型丰富，可自定义主题 |
| 数据查询 | DuckDB WASM | 浏览器端分析型数据库，执行 SQL |
| 数据格式 | Parquet | 随应用打包部署，高效压缩 |
| 后端 | Next.js API Routes | Serverless，处理 LLM 调用 |
| LLM | DeepSeek / OpenAI API | 自然语言 → SQL + 图表配置 |
| 部署 | Vercel | 免费，支持全栈，环境变量存 API Key |

### 数据流

1. 用户输入问题
2. 前端发送问题 + 当前数据集 schema 到 API Route
3. API Route 调用 LLM，传入 schema + 问题，要求返回结构化 JSON
4. LLM 返回 `{ sql, chart, insight }`
5. 前端用 DuckDB WASM 执行 SQL，获取数据
6. 前端用 ECharts 根据 chart 配置渲染图表
7. 展示洞察文字

### LLM 返回格式

```json
{
  "sql": "SELECT month, SUM(sales) as total_sales FROM orders GROUP BY month ORDER BY month",
  "chart": {
    "type": "line",
    "xField": "month",
    "yField": "total_sales",
    "title": "月度销售趋势"
  },
  "insight": "销售额在Q4达到峰值，同比增长23%，主要受双11和双12活动驱动。"
}
```

## 页面设计

### 页面 1：首页 / 数据选择页

- 品牌展示 + 一句话产品介绍
- 3 个内置数据集卡片（电商销售 / 用户行为 / 营销活动），点击进入分析
- "上传你的数据" 入口（CSV / Excel）
- 风格：简洁精致，参考 Linear / Notion 首页风格

### 页面 2：分析工作台（核心页面）

- **左侧**：数据概览面板（表结构、字段、数据预览）
- **主区域**：卡片式分析结果展示（每个分析 = 一张卡片：图表 + 洞察 + SQL）
- **底部**：输入框 + 预设问题推荐
- **首次进入**：展示 3-4 个推荐问题引导

### 页面 3：关于页

- 项目介绍、技术栈说明
- 个人简介和联系方式

### 设计原则：去 AI 味

- 不用对话气泡 → 用分析卡片
- 不用"AI 正在思考..." → 用"分析中..."
- 不用机器人头像 → 用图表/数据图标
- 整体像专业数据分析工具，不是套壳 ChatGPT
- 使用 frontend-design skill 打造高质量界面

## 内置数据集

| 数据集 | 表 | 约行数 | 典型分析场景 |
|--------|-----|--------|-------------|
| 电商销售 | orders, products, customers | ~5000 | 销售趋势、品类分析、客单价、RFM |
| 用户行为 | users, events, sessions | ~8000 | 留存分析、转化漏斗、活跃趋势 |
| 营销活动 | campaigns, channels, conversions | ~3000 | ROI 分析、渠道对比、归因分析 |

每个数据集附带 4-5 个预设问题。

## 错误处理

- SQL 执行失败 → 自动让 LLM 重新生成一次
- 查询结果为空 → 友好提示 "未找到匹配数据，试试换个问法"
- LLM 超时 → 超时提示 + 重试按钮
- 文件上传格式错误 → 提示支持的格式
